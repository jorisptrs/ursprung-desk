# The Desk — README

BRIEF.md is the source of intent. This file records how its ambiguities were ruled, one line each, plus run/deploy notes.

**Run:** any static server from the repo root — e.g. `python3 -m http.server 8123` → http://localhost:8123. ES modules refuse `file://`. Phone on the same wifi: `http://$(ipconfig getifaddr en0):8123` — the server binds all interfaces; `localhost` only resolves on the Mac itself.
**Deploy:** GitHub Pages, deploy-from-a-branch on `main` / root — a push is the deploy, and the empty `.nojekyll` at the root turns Jekyll off (the site is plain static files; Jekyll only adds a build that can fail and silently drops paths beginning with `_` or `.`). Verify a published build by fetching a file that exists **only** in it — `js/live.js` — never by the root's status code, which keeps returning 200 from the previous build. If a push lands and no `pages-build-deployment` run appears, check githubstatus.com before touching any setting: Actions and webhooks going degraded looks exactly like a broken repo.
**Views:** `/` deployed (opening pass, then live) · `/deposit.html` the hand door standing alone (the future QR target; its lays reach any table tab in the same browser) · `/?rig` filming rig (held, driver keys, no cursor) · `/?specimens` dev palette surface · `/?cursor=k` dev still at boundary k · `&flip=<id>` dev still with that card's back open · `&sheet` opens the deposit sheet over the view (with `?cursor=k` it is the sheet's byte-stable still; over `/` alone the pass moves beneath) · `&debug` clock narration + gesture-deadline canary · `&live` watches the MCP drop file (local only, D105) · `&tail=k` holds the stream's last k events back until `m` (D111). Only `/` and `deposit.html` are ever linked.
**Driver keys (rig):** `S`/`→` step · `←` back · `R` replay · `0` reset · `E` end · `L` live · `F` fullscreen. The faint `?` on the table lists gestures for every visitor (deployed: after the pass) and adds the keys on the rig.
**MCP (the deposit door, D104–D126):** on a new machine: `cd mcp && npm ci` once (Node 18+; `ffmpeg` too if you want recordings and takes cut — without it the door asks for a still instead). The server makes `drop/` itself on the first deposit. `.mcp.json` registers it for any Claude session opened in this repo; the manual equivalent is `claude mcp add desk -- node mcp/server.mjs`. The server writes `drop/stream.jsonl` (gitignored) and puts each card's trace in `drop/assets/` — a still verbatim, a recording drawn as its waveform, a take cut to a three-still strip (needs `ffmpeg` for the last two); a table open at `/?live` picks them up within a poll. `DESK_ROOT=/some/other/desk` points the door elsewhere.
**The room server (both networked doors, D129–D139):** `cd mcp && npm ci`, then `node mcp/room.mjs` from the repo root — one process on `:8080` (`DESK_PORT` to move it) owning the log, the assets and both doors. Seed the cohort with `node mcp/people.mjs add "E."` (`list` · `remove` alongside), then `node mcp/people.mjs qr` writes `drop/qr.html`: **one code for the whole room**, with a raw-IP fallback code beneath it (`.local` is the least reliable thing about an Android phone). Everyone scans the same one and taps their own name; the device remembers, and a laptop that claims is shown its own `claude mcp add` line. The projected table opens at `http://desk.local:8080/?rig&live` — the rig has no cursor and no `+`, so the room's doors are the only doors; press `l` to arm it. A phone deposits at `/deposit.html` — the same address for everyone, since the name is claimed rather than carried. The machine becomes `desk.local` with `sudo scutil --set LocalHostName desk` — Bonjour, no code. `mcp/npm audit` reports a Windows-only path traversal in a transitive Hono adapter the desk never serves static files through; the "fix" downgrades the SDK five minors, so it stands, assessed.
**The terminal shot (one take):** `: > drop/stream.jsonl && rm -f drop/assets/*` between takes, then open `/?live&tail=1` fullscreen and reload — the reload is to clear the table, not to revive the reader, which now survives a truncation on its own (D136). The pass deals everything but the meta card and rests live; `r` clears and `→` (held, it rushes) deals arrivals on cue. Say "put this on the desk" in a Claude session, confirm the card it proposes — it lands within a second. `d` pauses, `d` again re-deals the whole stream from empty (the deposit re-enacts in place); `m` lands the held meta card last.
**Test:** `node --test` (or `npm test`) from the repo root — dev-only, nothing in `tests/` or `package.json` ships. The MCP protocol suite skips itself until `mcp/npm install` has run. Two CDP end-to-end drives, neither part of `npm test`, both wanting the server up: `node tests/hand-e2e.mjs` (~60 s) and `node tests/mcp-e2e.mjs` (~40 s, also wants `mcp/node_modules`).

- D159 · (keeper's question 2026-07-23, settled by measurement) **The night's map drifts from last night's, and that is not a concession to spatial memory — it is the better map.** The question was whether to move each stack freely to its best place overnight or nudge it there over the retreat.   |---|---|---|

Two lines go with it: `front of the card · it stays closed` becomes just **front of the card** (whether a card opens is the card's business, not a caption's), and the dry line after picking a card back out of the deck goes entirely — the page changed under your eyes, which says it better than a sentence. What each register is for is said **on hover and at once**, not through the browser's own tooltip, which waits about a second — long enough that nobody deciding between three words ever sees it.
- D168 · (keeper's ruling 2026-07-23) **Writing carried in as a file is writing.** A poem dropped as a `.txt` lands on the page as its own words and fronts the card the way it would if it had been pasted; nobody should have to download a poem to read it. Anything the device calls text, or that wears a written extension, up to about 400 kB — past that it is a file, not a card. And the page is set the way the card will set it: a reading measure in the table's own serif, the title standing apart under a rule rather than merely bigger, and a long work scrolling inside the page rather than pushing the sheet down the screen. **What you write should look like the work while you are writing it.**
- D169 · (keeper's ruling 2026-07-23, narrows D11) **A card turns if there is more to see than the table can show at pile size.** A photograph, a crease pattern, a rendering, or words the pile-sized card had to clip: those open now, without any `detail` at all. That is not a disclosure — it is the card's own surface at reading size, which is what the front was already showing. What stays private is still `detail`, which is where the originals, the files and the links live, and **withheld is still withheld** (no `src`, no back). A summary is not a work: a waveform and a frame strip stay shut, because blowing up a summary shows nothing new — and that is also what keeps a withdrawal a withdrawal, which the seed's dish proved by failing the test when the rule was drawn too wide. A quest or a note on Claude that is nothing but its own sentence has nothing more to give, and stays shut.
- D170 · **A withdrawal takes the work off the table, not the person out of the room.** Retiring the only card that ever named someone was taking their studio with it — so a withdrawal shuffled the map for everyone else at the worst possible moment. Everyone the log has named keeps their place. With a night's arrangement standing — the castle's own case — a withdrawal now moves **nothing at all**; before the first arrangement, when the fold is solving the room itself, the space is given back as a nudge, which is the mirror of a new stack arriving (D154).
- D171 · **A thread's identity is its two ends, and an anchor's far end is the studio, never the studio's coordinates.** Keying on the place meant that redrawing the map destroyed every anchor and built new ones in their stead — so they blinked where they should have travelled. With that fixed the strings move with the piles: they are px and re-laid rather than re-flowed, and without animating them too they snapped to where the piles were going while the piles were still on their way, which was the one thing that made a rearrangement look wrong.

- D172 · **Whatever the registry does, the table hears about it as a fact in the log.** Registering someone already put their name on the wood; **renaming** now does too, so the new name takes a place at once instead of waiting for its first card. The old name keeps the place it was given — cards laid under it still carry it, and the log is not rewritten (D138). **Removal says what it leaves**: the registry is a list of who may claim a device, the table is a log, and a log is not un-written, so the name stays where it stands with nothing in it — said at the prompt rather than found out in the morning. Two things this shook loose: a test was reading the log by line number and broke the moment a roster line appeared between two deposits (it selects by event type now, which it should always have done); and the live reader was worth proving on non-deposit lines, since the room writes rosters into the very file `?live` polls — a reader that only understood cards would have dropped the room's own names, quietly, in a room, on the first night.

- D173 · (keeper's ruling 2026-07-23, amends D161) **A studio opened is that person's whole thread of thinking, collaborations included.** Someone asks a quest alone and finishes the work with somebody else: the quest stays on their pile and the work floats between the two, and until now reading the pair meant knowing to look in the gap between two piles. Opening a studio now gathers everything that person's hands are on, in the order it was made. The works made with somebody else are **ringed** rather than silently mixed in — they belong here as much as anywhere, and the mark says they belong elsewhere too, which is the whole point of a table where a work can be in two people's stories at once. A shared place opened on its own still shows only what those hands made together: it is a place two people keep, not a third person. Consequence: a tap on a visiting card reads it, rather than shutting this studio and opening the shared place it came from — the second beat belongs to whatever is currently spread.

- D174 · (keeper's ruling 2026-07-23) **A tap on the wood steps back one thing at a time.** The card in hand goes down first — **into the pile it came out of, still spread** — and only once nothing is in hand does the next tap shut the pile. Undoing two things with one tap is how you lose the place you were reading from, which on a table whose whole point is a person's thread of work is the worst thing a gesture can do. The same applies to the `?`: it now closes on a tap anywhere else, since a panel that will not go away when you look elsewhere is a panel you have to learn to dismiss.
