# The Desk — README

BRIEF.md is the source of intent. This file records how its ambiguities were ruled, one line each, plus run/deploy notes.

**Run:** any static server from the repo root — e.g. `python3 -m http.server 8123` → http://localhost:8123. ES modules refuse `file://`. Phone on the same wifi: `http://$(ipconfig getifaddr en0):8123` — the server binds all interfaces; `localhost` only resolves on the Mac itself.
**Views:** `/` deployed (opening pass, then live) · `/?rig` filming rig (held, driver keys, no cursor) · `/?specimens` dev palette surface · `/?cursor=k` dev still at boundary k · `&flip=<id>` dev still with that card's back open · `&debug` gesture-deadline canary. Only `/` is ever linked.
**Driver keys (rig):** `Space`/`S` step · `R` replay · `0` reset · `E` end · `L` live · `F` fullscreen. The faint `?` on the table lists gestures for every visitor (deployed: after the pass) and adds the keys on the rig.
**Test:** `node --test` (or `npm test`) from the repo root — dev-only, nothing in `tests/` or `package.json` ships.
**License:** code under MIT (see LICENSE). Media in `assets/` and the seeded content are demonstration stand-ins, not licensed for reuse.

## Decisions log

- D1 · Motion between events: the table is truly still at rest; each arrival nudges existing cards slightly edgeward, furniture-slow, then stillness again — strata are physically earned, no ambient drift.
- D2 · Image cards show the whole photograph in the parchment frame — a photograph is already a token; a tighter crop only when the depositor chooses one.
- D3 · Seed is invented demonstration content — no real people, no real deposits — and the outward deliverables (note, video, README) say so plainly, since no consent exists to imply.
- D4 · All seed assets are generated or public-domain placeholders under stable filenames in `assets/`, so real photographs can replace them later without touching `seed.json`.
- D5 · `detail` empty ≡ `detail` absent → the card has no back; the schema example's empty shape is illustrative, not a flippable back.
- D6 · Withheld excerpts keep their natural `form` with `text`/`src` omitted; the renderer draws the fallback (faint ruled lines, monospace dashes) — schema enum untouched.
- D7 · Waveforms ship pre-rendered in `excerpt.src`; the renderer never synthesizes a trace from the recording.
- D8 · Caption is optional; quest cards carry a title only.
- D9 · A failure card keeps the media of the attempted work where a trace exists; otherwise `note` media in the ashen register.
- D10 · `kind` and `media` compose freely; the seed simply uses sensible pairs.
- D11 · Tapping a backless card does nothing at all — the quietest option is the privacy signal.
- D12 · (amended 2026-07-21) On open, the deployed page runs one compressed pass — the four days assembling themselves from an empty field, quests first, ~12–18 s — then rests at full strata. No control shown; chrome-free. The rig instead starts held, with step/replay/reset under the driver.
- D13 · All motion passes through one queue — at most one thing moving, enforced globally; arrivals wait for flips and vice versa. Under a burst the log is right instantly and the view catches up — correct behavior, not a bug.
- D14 · A thread drawing itself is the arrival of a thread event, hence legal motion; threads then persist, fading with their older endpoint; `why` stays log-only in v0.
- D15 · Answered quests keep their faded state in v0; the thread itself is the payoff.
- D16 · v0 thread events are hand-authored in the seed and may only reference artifacts earlier in the stream.
- D17 · On the card the caption is authoritative; in the log the structured fields are; `practice` is filled on every artifact even though v0 renders it nowhere.
- D18 · Corrections to seed content are direct edits to `seed.json`; append-only semantics bind live sessions, not authoring.
- D19 · Live deposits (MCP, visitor drop) get a distinct id prefix and the current highest night; seed ids stay `a-###`.
- D20 · The voice skill is drafted before the seed content pass, despite §7 ordering it after core — captions can't precede the voice.
- D21 · `desk-plan.md` renamed to `BRIEF.md`; this folder is the repo root (§8's `desk/` name is cosmetic).
- D22 · Night-0 is the pre-retreat stratum (quests); nights 1–4 are the four days.
- D23 · Flip state is view-ephemera — inspection, not deposit — and is never logged.
- D24 · Placement and nudges are deterministic (id-hashed PRNG, stream-derived drift); no runtime randomness anywhere in the fold.
- D25 · Event table-times derive from stream order via a fixed spacing function; they are never stored in `seed.json`.
- D26 · The demo-driver conducts the clock; it never fabricates events.
- D27 · `js/` ES-module split with `desk.js` as entry — deviates from §8's single file so the pure core is node-testable; no bundler; the page is always served over http, never `file://`.
- D28 · Session drops sit in the same stream and re-enact on replay (Q31).
- D29 · Media adds `video` (a strip of stills or a single frame, depositor-chosen; attests, never plays) and `model` (a depositor-chosen still render; physical 3D enters as a photograph). Both draw through the existing image path — no new trace renderers.
- D30 · (amended 2026-07-21 by D72) Governing principle: the desk attests, it never performs *unbidden* — at rest nothing plays, spins, or scrolls; backs shelve files and link outward as quiet text, never embedded players; the renderer only ever draws excerpts; unlisted media reduce to a still, silent excerpt and a caption. One experience per back may be summoned by explicit gesture (D72).
- D31 · Schema enums extended: `media` = image | audio | video | text | code | fold | model | note; `excerpt.form` = crop | waveform | frames | sentence | lines | linework | render | words.
- D32 · Removal is retirement — an appended event the fold ceases to display (threads to it included); the stream itself is never rewritten.
- D33 · (amended 2026-07-21 by D65: photo → hand) Publishing boundary: the web demo appends to a per-tab fork that never merges; only three doors write the canonical stream (MCP deposit, hand path, curator); no fourth door — no save, no sign-in, no merge, ever.
- D34 · MCP door-1 contract: a deposit commits only on an explicit human go, stated in the tool description itself; v0 is a single `deposit` tool called once after in-session confirmation; the stage→confirm protocol is August scope; schema-violating deposits are rejected, never coerced.
- D35 · The rig runs behind any dumb static file server; "no backend" binds the deployed link only.
- D36 · Palette v1 and type are placeholders until the polish pass: warm near-black room and field, parchment cards, muted ink; system serif stack (Iowan/Palatino/Georgia) — no webfonts, so the rig works offline and nothing is fetched.
- D37 · Field geometry: the pool of light sits inset ~5% with a soft radial falloff; outside it stays unlit — no simulated wood, the real table supplies wood.
- D38 · The fold outputs normalized 0–1 table coordinates; the renderer maps them to pixels (portrait/phone mapping still open, Q33).
- D39 · Cards carry a faint contact shadow so they read as laid objects, not printed rectangles — eye-tunable at polish.
- D40 · Note-media cards suppress the separate title line when it would repeat the trace words verbatim — a quest is its own title, shown once.
- D41 · (amended 2026-07-21) Only deposits nudge the table (a thread draws, it doesn't displace); nudge = each arrival shoves every earlier card outward from its own landing point, one saturating-schedule increment scaled down with distance — the pile parts around what lands in it, lifetime-bounded per card.
- D42 · Card corners are hand-cut, not UI-rounded: per-corner elliptical radii (0.10–0.55em), the vertical radius drawn independently with lean clamped to 0.4–2.5× so a curve sometimes clearly favors one edge; hashed from the artifact id with a salt so they never shift placement — deterministic wear, stable across replays, applied to every card; no torn-edge deckle (craft kitsch).
- D43 · Overlap is welcome, legibility is not negotiable: deterministic relaxation in a canonical 1.6:1 table model pushes only the newer card, along the summed repulsion from every violated strip, until no card's title/caption strip is covered (whole card for note media); trace areas may be overlapped freely; constants re-tuned at the full seed pass.
- D44 · (amended 2026-07-21) `?specimens` dev view: 13 invented cards and two thread events covering every media, every kind register, both withheld fallbacks, and the thread paths (mid-stream draw, cross-night fade) — exercises all render paths, joins the screenshot verification loop, later the projection-calibration palette surface; never linked.
- D45 · Verification loop captured as the project skill `.claude/skills/run-desk/` (authored by hand — /run-skill-generator isn't available in this environment); every session verifies identically: tests, resource 200s, headless render, clean console, looked-at screenshot.
- D46 · (amended ×2 2026-07-21) One motion = one event's full consequence — the queue serializes gestures, not elements; the only reading under which D13 and D1 cohere. The pile begins giving way while the card is still travelling, rippling outward from the contact point (nearest first, starts ~30–62% of travel, all settled together at the gesture's end) — a slide the arrival causes, never a synchronized jolt at touchdown.
- D47 · The view is event-granular: always a settled fold state or tweening to the next; timeline.js owns the clock as a pure state machine (cursor, target, mode, tempo) — wall-clock never maps continuously into table-time.
- D48 · The DOM always holds the settled truth; animations are FLIP overlays (WAAPI, transform/opacity only) — cancelling any motion leaves a correct table, and resize needs no suppression machinery (positions in %, sizes in cqmin).
- D49 · (amended 2026-07-21) Relax runs on base placements; arrival shoves replay after and are schedule-bounded, so every card lives forever inside a fixed disc of its base — relax checks disc-inflated shapes and the floor holds at every boundary by construction; a laid card's between-event motion stays one schedule increment (<0.5% of field, machine-tested).
- D50 · A card enters from the dark rim along its placement ray (field rect inflated by one card diagonal, min travel 12% of the short side), opacity ramping only through the first quarter of travel; entry pose is view-side and salted-hash deterministic — no runtime randomness in motion either.
- D51 · Threads draw beneath the cards, trimmed to the two card edges so the draw-on is visible end to end; thread opacity = min of its endpoints' (D14 concretized — the schema never orders from/to by age); endpoints retarget once per gesture, never tracked per frame.
- D52 · Replay replays the full stream (§9's letter; D28) — meta lands last because it is last. Whether the rig needs a held-back-tail load so a live MCP deposit can precede meta in one unbroken take → MCP step.
- D53 · (amended ×4 2026-07-22) Driver behind `?rig` only; keys: Space/S step · R replay · 0 reset · E end · L live (the slow self-assembly is a B-roll shot and the MCP arm switch) · F fullscreen; repeats and modifier chords ignored on the rig. V retired with D54's resolution. The deployed page listens for exactly three keys — Space (next card in any mode; held down it keeps dealing in manual, never fast-forwards auto), D (pause ↔ resume the deal), `?` (help) — nothing else, ever.
- D54 · (resolved 2026-07-22) The pass and replay are strict-serialized — D13 stands unbent. The pipelined-rush variant was built, compared by eye, and removed outright: overlapping flight reads as a system performing, and the rush already comes from the compression. Removed, not shelved — the queue lost its overlap machinery too.
- D55 · No preload gate: excerpt fetches start at load, each arriving card waits only its own bounded decode (~300 ms) — one mechanism for every door; the first frame is never burned on a spinner-in-disguise.
- D56 · A hidden tab defers the pass to first visibility (boot hidden → empty pool, pass on `visible`); hiding mid-pass flushes to settled — arrivals are owed the greeting, returners aren't owed a rerun.
- D57 · `prefers-reduced-motion` on the deployed page: a stepped pass — instant placements on the same cadence, night beats kept; the preference is about motion, not narrative. `?rig` ignores it — the operator's keys are the consent to motion.
- D58 · Instant jumps (reset, end, replay's clear, hide-flush) flush the queue and write the settled state directly — scene changes are not motion.
- D59 · Every gesture resolves on `finished` or its duration + grace, whichever first — a throttled tab can never wedge the queue; a strict-mode deadline resolution logs a `?debug`-gated canary.
- D60 · Compressed pacing is budget-first (~15 s total, clamped inside D12's 12–18 s; per-event tempo derived, night beats and the opening rest carved out) — seed growth can never silently stretch the pass.
- D61 · A retire step fades its card and threads out inside the same gesture diff — removal is quiet, not theatrical.
- D62 · `?cursor=k` renders the fold at any boundary, motionless — the deterministic screenshot surface (never linked, like `?specimens`); headless virtual time freezes animation clocks, so moving pixels are judged by eye, stills by machine.
- D63 · The opening pass is replay entered at boot — one compressed policy, two entrances (D12).
- D64 · `?rig` hides the pointer (`cursor: none`) — nothing screen-like touches the wood, including the arrow.
- D65 · `provenance` names the door — who performed the gesture — never the medium; media already says what the thing is. Door 2 renamed `photo` → `hand` everywhere (enum `mcp | hand | curator`): the deposit page for any work from your own device — photographing analog work is its most common act, not its definition. The enum grows only if a door does, and new doors trigger the full consent machinery.
- D66 · Card geometry floors as a locked pair (width 120px·scale ⟺ its 0.075 font) via CSS `max()` once the field's short side drops under ~500 CSS px — browser zoom and small windows change how much table you see, never a card's shape; below the floor, per-card legibility outranks the D43 fit (projection, rig, and the video sit well above it).
- D67 · Fieldnotes keep to their corner (§9): the cartographer's pile sits lower-left, cascades up-right with the newest on top, never drifts and never sweeps; the pile may cover its own older notes — that is the register — while relax still protects it from every other kind, both ways.
- D68 · When relax's descent wedges in a local minimum, a deterministic hash-seeded spiral finds the newcomer the nearest free berth — the floor holds wherever a berth exists at scan resolution, and the every-boundary test says so if none does.
- D69 · (assets swapped back to placeholders 2026-07-22, rights unconfirmed — the ruling stands for their return under the same filenames, D4) The crane photograph feeds two cards: the crease-pattern card carries the CP as linework, and the backless work card "the fold, closed" carries the folded crane and threads back to R.'s quest — the answer no one announces.
- D70 · The demo disclosure lives on the meta card's back, in the desk voice ("a seeded demonstration — the people are invented, the works are stand-ins · the real table opens in October"); the table's face stays clean — D3 satisfied on the table itself.
- D71 · (video source withdrawn 2026-07-22; its play door returns with rights-cleared footage) Real assets land under clean names; audio excerpts are dev-rendered from the actual recording, one segment per card (D7's letter); the video excerpt is a three-still strip (D29); source files shelve on the backs; cards without a real asset keep their D4 placeholder under its stable name.
- D72 · (amends D30; covers websites and arbitrary/future formats) Every work's back may carry one experience, depositor-set: `detail.experience = { mode: "play" | "visit", src, demoSrc }`. `play` — the table performs it in place on an explicit gesture, one at a time, stillness after (audio, video, a looping visual). `visit` — the work runs elsewhere; the back carries the door: on the web demo a link out, at the castle a QR / short URL served from the desk's LAN store when the work is self-hostable. `demoSrc` is the compressed derivative for the deployed link; the castle serves full originals. Hard lines unchanged: nothing runs unbidden; the table itself never takes keyboard/mouse and never embeds another app's UI — interactive works are experienced on the visitor's own device or in the studio. Unknown formats resolve by one fork: can the table perform it silently-still by default? then play; otherwise visit.
- D73 · (amended ×2 2026-07-22) The flip: only backed cards turn (empty detail = no back = a silent tap, D11); tap anywhere on the card, tap again to turn back; one card in hand at a time; card taps are dead while the auto deal runs — an empty-table tap instead skips to the next card, the Space equivalent (D78); flip state is view-ephemera (D23). A swap is one exchange in one queue job: the open card starts down and the next lifts while it lands — one gesture, as an arrival and its nudges are one gesture (D46).
- D74 · (amended 2026-07-22) In hand a card grows 1.45×, straightens, lifts above the pile at full light, and slides just enough inward to sit aligned inside the pool's border; on close it returns exactly to its laid pose — picked up to read, put back where it lay. The back is quiet text on the sheet's reverse: small title (dropped on note media — the front already says it, D40) · the door word wearing a short amber underline, a thread run under the one word that answers a touch · shelved files and links as plain outward links · the note. No close affordance — the gesture is its own inverse.
- D75 · The play door performs once, in place, no controls: the door word becomes `still` (its own stop); tap again or turn the card back for stillness; one experience at a time globally; audio is sound alone, video fills the back and leaves it as it was. The deployed page plays `demoSrc` where one exists; the rig plays the original. The visit door opens a new tab; the table never moves.
- D76 · `?flip=<id>` boots a still view with that card's back open — the backs' deterministic screenshot surface (never linked; composes with `?cursor` and `?specimens`).
- D77 · (rewritten ×2 2026-07-22) The `?` sits on the table from the very start — warm white in the pool's quietest corner, on every mode including the opening pass, tappable by any visitor. Its help is plain language, not table prose, with one visual grammar throughout the desk: keyboard keys wear chips, tappable actions wear underlines (the card doors included). Dev views carry none. The deliberate soft spot in "no chrome inside the pool" stands: a question mark is a question.
- D78 · (amended ×2 2026-07-22) The deal control, deployed only: `d` — or its underlined row in the `?` help — is pause/play. Pausing snaps the card in flight placed and holds the table exactly where it is; nothing ever clears. Resuming deals the rest at pass cadence; at a full table, `d` re-deals from empty. Space — or a tap on empty table — advances to the next card in either mode: press-through when tapped, continuous dealing while held (manual only; holding never fast-forwards the auto deal). The rig keeps its own keys and no deal row.
